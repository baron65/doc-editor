## 2. 产品要求逐项核对

### 2.1 已有实现

- 目录不承载正文和发布状态；文档是唯一发布单元。
- 管理端树包含草稿，用户端树只包含已发布文档，并剪除没有已发布后代的目录。
- 文档 URL 使用不可变文档 ID，不依赖标题或路径。
- 同级草稿名称与线上名称有独立唯一约束。
- 目录最大四层、禁止移动到自身/后代、非空目录不可删除、已发布文档不可直接删除等后端规则已落地。
- 文档标题和正文使用编辑稿/线上稿双快照；发布和取消发布在事务内切换。
- draftRevision、treeRevision、publicationVersion 已进入接口模型。
- 图片和附件经后端代理上传至 `DocumentObjectStorage`；本地默认使用 MinIO S3 adapter。
- 草稿资源与线上资源使用 DRAFT/PUBLISHED 双引用；用户资源下载要求存在 PUBLISHED 引用。
- Mermaid 节点可以保存，用户 Reader 通过动态加载 Mermaid 渲染。
- 四个 Umi 路由已经存在且可直接访问。
- 2026-07-11 已补齐树标题过滤、目录折叠、当前文档父目录自动展开、文档中心默认打开第一篇已发布文档。

### 2.2 尚未开发或不满足设计

1. **自动保存剩余项**：已实现停止输入后保存、单飞队列、latest snapshot、失败快照保留、关闭页面提醒，以及管理端树内切换文档时的待保存/失败/冲突确认；浏览器后退等非页面入口仍依赖 `beforeunload`，需 E2E 复核。
2. **冲突处理剩余项**：前端已保留 Umi 非 2xx 响应的 HTTP 状态与业务码，并严格按 `DOCUMENT_VERSION_CONFLICT` 停止自动保存；仍需双会话 409 E2E。
3. **发布前置门禁剩余项**：已在预览/发布前校验 Mermaid、flush 保存队列，并在任何上传进行时统一禁用发布动作；后端再次校验全部引用资源为 READY。
4. **预览剩余项**：当前草稿已使用共享 Reader 和管理端资源地址预览；Reader 的代码高亮、页内目录等缺口会同时影响预览与用户阅读。
5. **附件剩余项**：已支持附件上传、结构化节点、管理端预览、用户下载、前后端白名单/大小前置提示；仍缺进度、替换、删除确认和失败重试。
6. **Markdown 粘贴剩余项**：已按强特征识别并支持标题、段落、基础 marks、列表、围栏代码、Mermaid 围栏和表格转换；仍需补完整 JSON round-trip fixture、失败时非阻塞提示和恶意 HTML/外链图片专项测试。
7. **富文本覆盖剩余项**：表格行列操作、Callout 四类型切换、代码语言与高亮/复制、图片替换/替代文本/说明编辑均已补齐；仍需富文本上下文操作的浏览器 E2E 和图片失败重试体验。
8. **树管理交互**：已有上下排序、显式跨目录移动与拖拽排序/移入目录；仍缺节点“更多”菜单，以及低频操作入口的一致收敛。
9. **状态表达**：管理树、草稿详情和编辑器已统一支持 `DRAFT / PUBLISHED / PUBLISHED_WITH_CHANGES`；同步已发布态禁止重复发布，只有待发布更新态显示“发布更新”。
10. **用户阅读体验**：最后发布时间、右侧页内目录、滚动章节高亮、上一篇/下一篇、代码语法高亮/复制和窄屏文档树抽屉均已实现；仍需补窄屏和滚动行为的浏览器 E2E 证据。
11. **异常页面**：下架/不存在已提供统一文案和返回文档中心入口；仍需补真实 404 E2E 证据。
12. **缓存与 HTTP 契约**：用户树和线上正文已按 `treeRevision/publicationVersion` 实现 ETag、`If-None-Match`、304 与 `Cache-Control: private, no-cache`；仍需真实运行接口验证发布/下架后 ETag 更新。
13. **安全响应头**：资源响应已设置并测试 `X-Content-Type-Options: nosniff`；后续还需系统验证 CSP、危险链接协议和 Mermaid SVG 安全策略。

## 3. 后端技术偏差与风险

### 3.1 契约偏差

- 树节点已增加 `nodeId/name/draftTitle/publishedTitle/publishState`，并暂时保留 `id/title/published` 兼容字段；前端已经消费三态字段，后续迁移企业仓库时可统一删除兼容字段。
- 编辑稿详情已增加 `parentId`、`draftTitle`、`publishState`、`publishedTitle`、`publishedAt`、`draftUpdatedAt`；线上详情已增加仅来自 PUBLISHED 引用的资产元数据表，管理端详情仍沿用独立资源接口。
- 已发布详情已返回 `publishedAt` 并由用户 Reader 展示。
- 创建目录/文档已支持设计中的 `targetIndex` 并重排同级节点；未传时仍追加到末尾。
- 创建接口已返回 201 和 `nodeId/documentId`，同时暂留 `id` 兼容字段；业务异常已按详细设计拆分稳定错误码和 HTTP 状态。

### 3.2 并发与事务风险

- 草稿保存已改为数据库条件更新 `WHERE draft_revision = ?` 并校验影响行数，消除了两个并发保存同时通过内存预检后的静默覆盖；仍需 MySQL 双连接集成测试证明真实事务行为。
- treeRevision 已改为事务末尾条件递增，竞态失败会抛出 `TREE_VERSION_CONFLICT` 并回滚整次树变更；仍需 MySQL 双事务集成测试证明节点写与 revision 回滚。
- 发布改为数据库内列到列快照复制，并同时条件校验 draftRevision/publicationVersion；下架按 publicationVersion 条件更新。三类竞态分别返回稳定错误码，仍需数据库故障注入证明旧线上稿完整。
- 应增加 MySQL 集成测试证明冲突、回滚、唯一索引和发布故障时线上稿不变，现有 Mockito 单元测试不能证明数据库并发语义。

### 3.3 资源与安全

- 资源 READY 状态、引用校验与 `nosniff` 已存在；上传新增可配置扩展名、声明 MIME 和文件签名联合校验，默认拒绝 SVG、HTML、可执行附件及伪装文件。
- 清理任务已存在代码骨架，仍需验证 UPLOADING 超时、DELETING 抢占、对象不存在幂等和多实例竞争。
- 正文后端目前主要做结构/大小校验；用户端链接协议和最终渲染安全仍需系统测试。

## 4. 前端技术偏差与风险

- `DocumentEditorShell` 同时承担上传、保存、发布、下架和状态展示，继续扩展会迅速失控。扩展配置和保存协调器已经拆出；下一步继续拆发布门禁、工具栏与预览壳。
- Reader 是手写递归渲染，尚未覆盖完整 schema；编辑器与 Reader 的“共享 Schema”目前只是共享 JSON 类型，不是共享节点定义和行为。
- Umi `useRequest` 默认 `formatResult` 会再次取 `.data`，而 service 已经 unwrap。2026-07-11 已通过显式 passthrough 修复四个页面，后续所有 service hook 必须沿用同一约定。
- 当前没有前端测试命令。已为树模型增加 Node 内置测试，但仍需建立正式、稳定的组件测试入口并覆盖保存队列、Markdown、Mermaid gate、Reader 和路由行为。
- 用户端目前可能打包进管理编辑器依赖与否尚无 bundle 报告，不能满足完成定义。
