# 文档中心综合实现计划

> 本计划合并并取代以下增量计划的执行入口：块级上下文工具栏、文档中心单入口、表格编辑交互、表格插入与整表删除。原计划文件保留作为历史设计记录，不再单独推进。

## 目标

在现有 React + Umi + Tiptap 文档中心中完成一套一致的单页工作台、块级气泡工具栏和表格编辑体验，同时保证草稿/发布模型、Markdown 粘贴、预览和用户端渲染不回退。

## 不变约束

- 不新增实时协作、评论、分享、导出、历史版本或文档级权限模型。
- 不修改后端接口、数据库字段和既有草稿/线上双快照协议，除非结构化内容校验确有必要补充已声明属性。
- 不使用 `window.prompt` 或 `window.confirm`；破坏性操作使用公共确认弹框。
- 不增加第三方依赖，不执行 Git 暂存、提交、重置或清理。
- 表格块气泡工具栏不提供插入列、删除列、插入行、删除行；这些能力归属表格专用工具条和边缘控件。

## 阶段 1：统一页面入口与状态流

**涉及：** `frontend/src/pages/document-center/`、`frontend/src/pages/admin/document-center/`、路由配置及单入口测试。

- [ ] 用户端将树、正文、上下篇导航和错误态合并到 `index.tsx`，同时支持根路径和带 ID 路径。
- [ ] 管理端将文档树管理与编辑器合并到 `index.tsx`，删除 `detail.tsx`。
- [ ] 管理端区分 `selectedTreeNode` 与 `activeDocumentId`；点击目录不切换正文，点击文档才更新 URL 和详情请求。
- [ ] 保留新建、重命名、删除、排序、移动、拖拽和未保存离开确认。
- [ ] 树刷新、保存、发布、下架后及时更新状态标签，但不卸载当前编辑器。
- [ ] 验证四种历史 URL、刷新、默认文档跳转和空状态。

## 阶段 2：块级气泡工具栏与编辑态一致性

**涉及：** `BlockContextToolbar`、块定位模型、快捷键模型、格式化扩展、Reader 渲染和全局样式。

- [ ] 块手柄按鼠标坐标精确定位当前段落、标题、列表项、引用、代码块、表格或特殊节点；目标节点在手柄/菜单激活期间高亮。
- [ ] 空段落显示 `+`，非空块显示当前类型图标；菜单优先向正文外侧展开，宽度受限并在视口内滚动。
- [ ] 顶部转换区横向展示正文、H1～H5、无序列表、有序列表、任务清单、引用和代码块；其余插入和节点操作纵向排列。
- [ ] 增加加粗、斜体、下划线、删除线、字号、颜色、缩进、对齐、链接、复制节点和删除节点；复制仅写入剪贴板。
- [ ] 缩进/对齐、颜色、字号采用右侧级联子菜单；菜单 hover 时不覆盖主菜单，不使用返回式替换。
- [ ] 所有可见工具注册编辑器范围内快捷键，并显示 macOS 与 Windows/Linux 文案。
- [ ] Markdown 粘贴优先走 Markdown 解析，失败后再走富文本/纯文本；不能将整段 Markdown 粘贴为代码块。
- [ ] 对齐、缩进、字号、颜色使用安全枚举和范围，编辑态、预览态、用户端共用渲染语义。
- [ ] Mermaid 仅由明确的 Mermaid 节点/语言标识触发渲染，普通代码块继续按代码展示。

## 阶段 3：表格编辑与插入删除

**涉及：** `TableContextToolbar`、`TableSizePicker`、`TableKeyboardExtension`、表格模型、Schema 配置和样式。

- [ ] 保留 Tiptap 原生 Table Schema，开启列宽拖动、最小列宽和多单元格选区反馈。
- [ ] 表格专用工具条提供行、列、单元格、表格四组菜单：新增/删除行列、合并/拆分、表头切换和整表删除。
- [ ] 表格边缘显示行列选择句柄、指定位置插入按钮和直接删除按钮；目标行列必须可明确选中。
- [ ] 块气泡工具栏的“表格”入口打开 1～20 的飞书式网格尺寸选择器，支持 hover 拖选、边缘扩展和数字输入，默认带首行表头。
- [ ] 点击表格块手柄建立表格 NodeSelection；仅该选择响应 Backspace/Delete 删除整表，单元格编辑和 CellSelection 保持原行为。
- [ ] 表格整表删除菜单继续使用公共确认弹框；键盘删除遵循通用编辑器删除选中节点行为。
- [ ] 工具条、边缘控件和块气泡工具栏的职责边界写入回归测试，防止行列菜单重新出现在块气泡中。

## 阶段 4：验证与回归

- [ ] 纯函数测试：块定位、图标映射、快捷键、尺寸选择、表格上下文、整表 NodeSelection。
- [ ] 静态 UI 测试：单页入口、固定布局、级联菜单、气泡工具栏职责边界、表格入口和快捷键。
- [ ] 运行全部 TypeScript 测试：`pnpm exec tsx --test $(rg --files src | rg '\\.test\\.ts$')`。
- [ ] 运行全部静态测试：`node --test $(rg --files src | rg '\\.test\\.mjs$')`。
- [ ] 运行 `pnpm typecheck`、`pnpm build` 和 `git diff --check`。
- [ ] 浏览器做非破坏性验证：根路径/带 ID 页面、文档树固定、编辑操作栏固定、尺寸选择器、表格上下文工具条和整表节点选中；不修改现有业务文档数据。

## 完成标准

1. 总产品文档中的状态、页面入口、编辑器、表格和验收标准与实现边界一致。
2. 两端均只有一个页面入口组件，旧 URL 继续可用。
3. 块级气泡工具栏不包含表格行列增删，表格专用工具条仍完整可用。
4. 表格支持网格/数字插入、指定位置行列操作和整表键盘删除。
5. Markdown、Mermaid、格式化属性在编辑态、预览态和用户端表现一致。
6. 全量测试、类型检查和生产构建通过。
