# 块级上下文工具栏设计

## 目标

将管理端编辑器顶部常驻的大工具栏替换为类似飞书的块级上下文操作：鼠标悬停内容块时在左侧显示当前块类型入口，空段落显示 `+`，悬停或点击入口后展开工具箱；选中文字时显示紧凑的行内格式菜单。

## 交互

- 非空块显示块类型缩写：正文 `T`、标题 `H1`～`H5`、列表 `•/1.`、引用 `“`、代码 `{}`、表格 `▦`、图片 `▧` 等。
- 空段落显示 `+`。
- 块入口与菜单构成同一悬浮区域，鼠标可从入口移动到菜单，不会意外关闭。
- 列表按列表项逐行定位，不把整组列表误认为同一行。
- 工具箱顶部的块类型转换采用飞书式紧凑图标区：正文、H1～H5、列表、编号、引用和代码块按五列两行排列；插入、提示块和上下文操作继续纵向排列。
- 工具箱宽度为 `240px`；桌面端优先在手柄左侧展开，避免遮挡正文，左侧空间不足时才在右侧展开。
- 正文、列表、编号、引用和代码块使用语义化 SVG 图标，标题继续使用 `H1`～`H5` 文本图标。
- 工具箱超出视口时内部滚动。
- 块菜单只包含本产品已有能力：正文、标题、列表、编号、引用、代码块、分割线、表格、四类提示块、图片、附件和 Mermaid；不增加评论、分享、翻译等能力。
- 选中文字时显示紧凑浮动菜单，提供加粗、斜体、下划线和链接。
- 表格和图片的上下文操作保留在块菜单中。
- 每个可见工具都展示并注册快捷键；macOS 使用 `⌘/⌥/⇧`，Windows/Linux 使用 `Ctrl/Alt/Shift`，且快捷键只在编辑器聚焦时生效。
- 鼠标进入块手柄或工具箱时，以浅蓝背景和左侧强调线高亮目标节点；离开后恢复，不修改文档内容。
- 主菜单增加“缩进和对齐”“颜色”二级菜单，以及“复制节点”“删除节点”操作。
- 普通段落和标题支持 0～6 级缩进，每级 `24px`；列表的增加/减少缩进使用列表层级命令。
- 对齐支持左对齐、居中、右对齐和两端对齐。
- 文本颜色使用固定安全色板：默认、灰、红、橙、绿、蓝、紫；不接受任意 CSS 颜色值。
- “复制节点”在当前节点之后创建完整副本；“删除节点”立即删除当前节点，可通过编辑器撤销恢复。
- 原顶部内容工具栏删除；发布操作栏不变。

## 技术方案

- 使用 ProseMirror `posAtCoords` 将鼠标视口坐标映射到文档位置，再解析当前顶层块或列表项；不增加 DragHandle 依赖，也不实现拖拽排序。
- 块工具栏作为独立 React 组件，输入 editor 与现有上传/插入回调。
- 菜单打开时不抢占编辑器焦点；执行类型转换时才把选区移动到目标块。
- 插入操作保存目标块的精确插入位置，图片和附件完成异步上传后仍插入原目标位置。
- 行内格式使用基于选区坐标的轻量浮动菜单，兼容项目当前 TypeScript 解析配置。
- 快捷键模型集中维护按键匹配和平台化文案；块菜单组件只负责把快捷键映射到现有 Tiptap 命令及上传回调。
- 节点高亮通过 ProseMirror node decoration 临时渲染样式，不进入 Tiptap JSON，也不触发自动保存。
- 对齐与缩进通过自定义全局节点属性持久化；颜色通过自定义 `textStyle` mark 持久化。后端只接受已声明的枚举、范围和安全色板。
- 用户端 Reader 读取同一属性和 mark 渲染，不直接信任持久化 CSS 字符串。
- 编辑器左侧增加手柄空间；窄屏仍允许工具箱在正文区域内显示。

## 快捷键

| 工具 | macOS | Windows / Linux |
|---|---|---|
| 加粗、斜体、下划线、链接 | `⌘B`、`⌘I`、`⌘U`、`⌘K` | `Ctrl+B`、`Ctrl+I`、`Ctrl+U`、`Ctrl+K` |
| 正文、H1～H5 | `⌘⌥0`、`⌘⌥1`～`⌘⌥5` | `Ctrl+Alt+0`、`Ctrl+Alt+1`～`Ctrl+Alt+5` |
| 列表、编号 | `⌘⇧8`、`⌘⇧7` | `Ctrl+Shift+8`、`Ctrl+Shift+7` |
| 引用、代码块 | `⌘⇧B`、`⌘⌥C` | `Ctrl+Shift+B`、`Ctrl+Alt+C` |
| 分割线、表格 | `⌘⌥-`、`⌘⌥T` | `Ctrl+Alt+-`、`Ctrl+Alt+T` |
| 图片、附件、Mermaid | `⌘⌥P`、`⌘⌥F`、`⌘⌥M` | `Ctrl+Alt+P`、`Ctrl+Alt+F`、`Ctrl+Alt+M` |
| 信息、注意、成功、危险提示 | `⌘⌥⇧1`～`⌘⌥⇧4` | `Ctrl+Alt+Shift+1`～`Ctrl+Alt+Shift+4` |
| 表格列/行操作、删除表格 | `⌘⌥→`、`⌘⌥⇧→`、`⌘⌥↓`、`⌘⌥⇧↓`、`⌘⌥⌫` | 对应 `Ctrl+Alt` 组合 |
| 替换图片、替代文本、图片说明 | `⌘⌥R`、`⌘⌥E`、`⌘⌥G` | 对应 `Ctrl+Alt` 组合 |

## 验证

- 纯函数测试覆盖节点类型到图标、空段落到 `+` 的映射。
- 纯函数测试覆盖快捷键匹配和跨平台显示。
- 源码回归测试覆盖紧凑图标区、快捷键监听和节点高亮接入。
- TypeScript 检查和 Umi 构建必须通过。
