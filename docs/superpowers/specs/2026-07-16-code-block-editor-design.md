# 文档中心代码块编辑交互设计

- 日期：2026-07-16
- 状态：已确认
- 范围：代码块语言选择、语言搜索、自动换行、复制

## 1. 背景与目标

当前管理端代码块直接使用 Tiptap 默认 `codeBlock` 编辑形态，语言需要在插入时通过弹框输入，插入后缺少直观的语言切换、自动换行和复制能力。目标是在不引入 Monaco 或 CodeMirror 的前提下，参考飞书代码块的操作方式，为每个代码块提供节点内工具栏，并保证编辑稿、预览稿和用户端读取同一语言属性。

本次不包含行号、代码块折叠、代码执行、代码格式化和语言服务。

## 2. 方案决策

采用 Tiptap React NodeView 扩展现有 `codeBlock` 节点。

- `NodeViewWrapper` 承载代码块容器和不可编辑工具栏。
- `NodeViewContent` 继续承载可编辑代码正文，不改变 ProseMirror 的选区、撤销和粘贴能力。
- 通过 `updateAttributes({ language })` 更新 `codeBlock.attrs.language`，语言随 Tiptap JSON 保存。
- 不引入新的编辑器内核，避免双编辑器选区、快捷键和内容同步问题。

## 3. 交互设计

代码块顶部显示固定工具栏：

1. 左侧显示“代码块”标识。
2. 右侧显示当前语言。点击后打开下拉面板。
3. 下拉面板顶部提供搜索框，按语言名称和别名过滤。
4. 当前语言使用勾选和强调色标识；选择语言后立即关闭面板并更新语法类型。
5. “自动换行”按钮切换当前编辑器会话中的代码展示方式。开启时使用 `white-space: pre-wrap` 和单词断行；关闭时保留横向滚动。
6. “复制”按钮复制完整代码文本，成功后短暂显示“已复制”；失败时显示明确的失败状态。

点击工具栏、搜索框或语言项时不改变代码正文内容。工具栏设置 `contentEditable=false`，避免操作被写入文档。

## 4. 语言模型

建立共享语言目录，至少包含当前高亮器已注册的语言：

- Plain Text
- Bash / Shell
- CSS
- HTML / XML
- Java
- JavaScript
- JSON
- Markdown
- Python
- SQL
- TypeScript
- YAML

语言目录包含稳定值、展示名称和搜索别名。例如 `javascript` 可被 `js` 搜索到，`bash` 可被 `shell` 和 `sh` 搜索到。

文档 JSON 只保存稳定语言值。遇到历史数据或暂未注册的语言时，不丢弃原值：编辑态仍显示该值，阅读端回退为纯文本，不执行错误的语法高亮。

Mermaid 仍由独立 Mermaid 节点处理，不在普通代码块语言选择中提供，避免普通代码块和流程图节点语义混淆。

## 5. 组件边界

### 5.1 CodeBlockExtension

继承现有 Tiptap `CodeBlock` 节点，保留 `language` 属性、HTML 解析和序列化规则，新增 React NodeView。

### 5.2 CodeBlockNodeView

负责编辑态工具栏、语言搜索面板、自动换行状态、复制反馈和代码正文容器。语言变更通过 NodeView 提供的 `updateAttributes` 回写节点。

### 5.3 codeLanguages

共享语言元数据和语言规范化函数，供编辑态语言列表与用户端高亮器共同使用，避免两侧支持范围和别名漂移。

### 5.4 CodeBlock 阅读组件

继续负责静态高亮和复制。它读取同一个 `language` 属性，并复用语言规范化逻辑。本次不要求用户端提供语言切换和自动换行开关。

## 6. 数据流

1. 用户在代码块语言菜单中选择语言。
2. NodeView 调用 `updateAttributes({ language })`。
3. Tiptap 产生更新事务并更新文档 JSON。
4. 现有自动保存逻辑保存草稿。
5. 预览和用户端从 `codeBlock.attrs.language` 读取语言并高亮。

自动换行属于编辑会话 UI 状态，不写入文档 JSON，也不触发自动保存。

## 7. 异常与边界处理

- 剪贴板 API 不可用或复制失败时，按钮显示“复制失败”，不静默失败。
- 搜索无结果时显示“未找到匹配语言”。
- 切换语言不修改代码正文、不重建代码块、不移动选区。
- 点击代码块外部或按 Escape 关闭语言面板。
- 长代码关闭换行时由代码正文区域横向滚动，不能撑破文档容器。
- 未识别语言按纯文本显示，但原始 `language` 属性保留。

## 8. 验收标准

1. 新建代码块不再要求通过输入弹框指定语言，默认使用 Plain Text。
2. 每个代码块顶部均可查看和切换语言。
3. 语言列表支持按展示名和别名搜索。
4. 语言切换后，草稿 JSON 中的 `attrs.language` 正确更新并能自动保存。
5. 重新打开文档、用户视角预览和用户端阅读时语言保持一致。
6. 自动换行可即时切换，关闭时长行可横向滚动。
7. 复制成功和失败均有可见反馈。
8. 工具栏操作不会插入文字、改变代码正文或破坏撤销/重做。
9. 历史代码块和未知语言代码块可以正常加载，不产生运行时错误。

## 9. 测试范围

- 语言目录：规范化、别名搜索、未知语言回退。
- NodeView：语言选择回写属性、搜索过滤、换行切换、复制反馈。
- 编辑器集成：代码块 JSON 保存与重新加载、编辑内容不受工具栏操作影响。
- 阅读端回归：支持语言高亮、未知语言纯文本回退、复制功能。
- 构建验证：TypeScript 类型检查和生产构建通过。
