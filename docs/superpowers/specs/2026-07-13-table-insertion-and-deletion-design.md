# 表格插入与整表删除交互设计

日期：2026-07-13

## 目标

完善管理端富文本编辑器的表格插入与删除体验，使运营可以直观选择新表格尺寸，并明确区分单元格编辑、行列操作和整表删除。

## 插入表格

- 点击块工具中的“表格”后打开尺寸选择面板。
- 面板默认展示 5×5 网格；鼠标移动到当前边缘时继续扩展，最大 20×20。
- 鼠标经过的矩形区域实时高亮，并显示“插入 N × M 表格”。
- 点击网格单元完成插入，调用 Tiptap `insertTable({ rows, cols, withHeaderRow: true })`。
- 面板下方保留行数、列数数字输入，范围均为 1～20，供精确输入和键盘操作。
- 数字输入与网格选区双向同步；点击“插入表格”按钮执行输入尺寸。
- 点击面板外或按 Escape 关闭面板，不修改正文。

## 块气泡工具栏

- 移除块句柄气泡工具栏中“表格操作”分组里的插入列、删除列、插入行、删除行。
- 表格专用上下文工具栏继续提供行、列、单元格和整表操作。
- 行列的插入、选择和删除继续由表格边缘控件负责：边界 `+` 精确插入，行列句柄选中，红色 `×` 删除。

## 整表选择与键盘删除

- 表格块句柄支持选中整张表格，编辑器状态使用表格节点的 `NodeSelection`。
- 仅当当前选择是表格节点的整表选择时，Backspace 或 Delete 才删除整张表格。
- 光标位于单元格内、文本被选择或多个单元格被选择时，Backspace/Delete 不触发整表删除，继续使用编辑器原有行为。
- 键盘整表删除不显示确认弹框，保持与通用编辑器删除选中节点的行为一致。
- “表格”菜单中的“删除整张表格”继续使用公共确认弹框。

## 组件边界

- `TableSizePicker`：维护网格悬停尺寸、扩展尺寸和数字输入，只通过 `onInsert(rows, cols)` 输出结果。
- `BlockContextToolbar`：打开尺寸面板，并在确认后执行 Tiptap 插入命令。
- `TableContextToolbar`：继续负责表格专用菜单和表格边缘行列控件。
- 表格键盘扩展：识别表格 `NodeSelection`，注册 Backspace/Delete 快捷键并执行整表删除。

## 验收标准

- 可通过拖选网格插入 1×1 至 20×20 的表格。
- 可通过数字输入插入相同范围内的表格。
- 块句柄气泡工具栏不再出现行列增删操作。
- 表格边缘控件仍能在指定位置插入或删除行列。
- 选中整张表格后 Backspace 和 Delete 均可删除表格。
- 单元格内按删除键不会误删整张表格。
- 类型检查、编辑器单元测试、UI 静态测试和前端生产构建通过。
