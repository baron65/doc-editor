# 文档表格编辑交互设计

## 目标

在不改变现有 Tiptap JSON、后端接口和保存流程的前提下，为管理端表格补齐接近飞书的常用编辑能力。范围控制在运营文档编排所需的行列、单元格和表格结构操作，不增加公式、排序、单元格颜色等高级能力。

## 方案

采用方案 A：保留 Tiptap 3.27.3 内置 Table 扩展，新增轻量的 `TableContextToolbar`，调用编辑器原生命令完成结构操作。不开发表格 NodeView，不修改文档 Schema。

## 交互设计

1. 光标或单元格选区进入表格时，在表格顶部显示紧凑悬浮工具条；离开表格、切换文档或打开预览时隐藏。
2. 工具条不能遮挡当前编辑单元格，并随表格在视口中的位置更新。
3. 工具条提供四组级联菜单：
   - 行：上方新增、下方新增、删除当前行。
   - 列：左侧新增、右侧新增、删除当前列。
   - 单元格：合并所选单元格、拆分当前单元格。
   - 表格：切换首行表头、删除整张表。
4. 命令不可执行时禁用对应菜单项，例如单一普通单元格不能合并，未合并单元格不能拆分。
5. 开启列宽拖动，单元格最小宽度为 80px；拖动期间显示清晰的蓝色列边界反馈。
6. 保留 Tiptap 原生多单元格选区，并增强选区背景和边框反馈。
7. 删除行、列或整表直接执行；删除整表使用项目公共确认弹框，不使用 `window.confirm`。

## 组件边界

- `TableContextToolbar`：监听编辑器 selection/update/blur，计算表格位置，展示菜单并执行命令。
- `tableContextModel`：从 ProseMirror selection 中解析当前是否处于表格、命令是否可用，保持定位与状态逻辑可单测。
- `documentSchemaExtensions`：仅调整 Table 配置，开启 `resizable`、设置拖动手柄和最小列宽。
- `global.css`：定义悬浮工具条、列宽手柄、单元格选区和窄屏适配样式。
- `DocumentEditorShell`：挂载工具条并复用现有公共确认弹框。

## 数据与保存

所有操作直接产生 Tiptap transaction，触发现有 `onUpdate`，继续由现有自动保存协调器保存。列宽保存在 TableCell/TableHeader 的 `colwidth` 属性中；无需新增接口、数据库字段或后端 Schema 类型。

## 边界与异常

- 编辑器未初始化、命令不可用或选区离开表格时，不执行操作并隐藏/禁用控件。
- 工具条定位超出视口时自动收敛到可视区域。
- 删除最后一行或最后一列等价于删除整张表，并经过整表删除确认。
- 表格横向溢出时由现有 `.tableWrapper` 独立滚动，不撑破编辑区。

## 验收标准

1. 可以在当前行上下、当前列左右新增，并删除指定行列。
2. 可以拖动列边界调整宽度，刷新后列宽仍存在。
3. 多选单元格后可以合并，合并单元格可以拆分。
4. 可以切换首行表头和删除整表。
5. 工具条不遮挡编辑内容，滚动和切换选区时定位正确。
6. 所有操作触发现有自动保存，预览和用户端表格结构一致。
7. 新增模型、组件行为和配置测试，前端全量测试、类型检查和构建通过。
